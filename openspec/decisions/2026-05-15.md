## 00:21 — `gh` CLI 完整自動化首次 deploy 流程可行

不需要手點 GitHub UI，全部走 CLI：

```bash
gh repo create <owner>/<repo> --public --description "..." --source . --remote origin --push
gh api repos/<owner>/<repo>/pages -X POST -f build_type=workflow
gh api repos/<owner>/<repo>/actions/permissions/workflow -X PUT -F default_workflow_permissions=write -F can_approve_pull_request_reviews=false
gh run watch <run-id> --exit-status
```

未來 fork / 新 project 起 GH Pages 都可以複製這個 pattern。需要 `gh auth scope: repo + workflow`（auth status 可確認）。
