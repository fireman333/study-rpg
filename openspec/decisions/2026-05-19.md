## 00:50 — 二階題庫附圖 truncation：cross-page render fix

**Context**: commit `f03a1fd` (2026-05-18) batch re-extract 162 broken question images，commit message 號稱「173 broken → 11 still under 30 KB（94% recovered）」並把剩 11 張當邊緣 case 不處理。本 session QA 揭露 11 張中其實有 **3 張仍是 truncated**：

| qid | before | after fix |
|---|---|---|
| `106-1-醫學三-內科-Q11` | 2732×361 (581 KB) 中段橫切片 | 923×1072 完整 chest X-ray |
| `115-1-醫學五-骨科-Q74` | 312×89 (17 KB) label-bar sliver | 1824×994 hip X-ray + T1WI MRI |
| `115-1-醫學四-神經內科-Q75` | 396×79 (20 KB) sliver | 2165×990 coronal T2 + sagittal T1 brain MRI |

**Root cause**:
- Q74 / Q75：stem 落在 page 底部（y≈731/764，page height 842），figure 在**下一頁**的頂部。原 `tools/extract-medexam2-images.py` 沒處理 figure 跨頁的情況，只在 stem page 找 embedded image → 取到 caption bar fragment
- Q11：figure 由多個 embedded image tile 拼成（page 1 從 y=504 起多列），原 script 只取第一個 tile → 中段橫切片

**Fix approach** (in `/tmp/medexam2-fix-3-truncated.py`):
1. 定位 stem marker + next-Q marker（允許跨頁）
2. union 兩個 marker 之間的所有 embedded image bbox
3. 用 `page.get_pixmap(clip=bbox, matrix=300dpi)` render page **slice**（不是抽 embedded image xref）
4. 跨頁時 vstack 兩個 slice

關鍵差異：page slice render 對 embedded image tile / 多 xref 自動 immune，因為直接從 PDF raster page 取。

**Lesson 給未來補圖任務**:
- aspect ratio > 4.5:1 或單邊 < 100px 是 truncation 的決定性 signal（不該只看 file size < 30 KB）
- 補圖 script 一律走 page-slice render，不要再用 embedded image xref extraction
- "Skip remaining edge cases" 結論在沒做 visual QA 前不可下；本次 11 張剩餘 9/11 確實正常但 3/11 是 broken，純看數字會 miss
